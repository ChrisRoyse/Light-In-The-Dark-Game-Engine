# new-capabilities.txt — reverse-closure whitelist for jassgen -revclosure (#260).
#
# Reverse closure: every EXPORTED func/method in litd/api must trace to either
#   (a) a goMapping symbol in api-manifest.json (a deduplicated JASS port), or
#   (b) a rule in this file (a LitD-original capability or structural plumbing).
# Anything in neither set is unaccounted scope and FAILS the gate. Exported
# types/vars (data structures, options/event structs, enums) are out of scope —
# this gate governs the BEHAVIORAL surface (callable verbs), not data shapes.
#
# Rule forms (first match wins; '#' starts a comment; blank lines ignored):
#   suffix  <.Method>     any "<Type>.<Method>" method with this name
#   prefix  func <Pfx>    any package func whose name starts with <Pfx>
#   exact   <Symbol>      one exact "Type.Method" or bare "Func"
#
# Grandfathered surface as of #260: the entries below are the known new-API verbs
# at M2. A NEW export not covered here (or by a new manifest mapping) trips CI —
# which is the point: it forces every added verb to be either a JASS port or a
# consciously-declared new capability.
