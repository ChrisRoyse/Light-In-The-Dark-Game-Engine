# new-capabilities.txt — reverse-closure whitelist for jassgen -revclosure (#260).
#
# Reverse closure: every EXPORTED func/method in litd/api must trace to either
#   (a) a goMapping symbol in api-manifest.json (a deduplicated JASS port), or
#   (b) a rule in this file (a LitD-original capability or structural plumbing).
# Anything in neither set is unaccounted scope and FAILS the gate. Exported
# types/vars (data structures, options/event structs, enums) are out of scope —
# this gate governs the BEHAVIORAL surface (callable verbs), not data shapes.
#
# Rule forms (first match wins; '#' starts a comment; blank lines ignored):
#   suffix  <.Method>     any "<Type>.<Method>" method with this name
#   prefix  func <Pfx>    any package func whose name starts with <Pfx>
#   exact   <Symbol>      one exact "Type.Method" or bare "Func"
#
# Grandfathered surface as of #260: the entries below are the known new-API verbs
# at M2. A NEW export not covered here (or by a new manifest mapping) trips CI —
# which is the point: it forces every added verb to be either a JASS port or a
# consciously-declared new capability.

prefix  func New

# combat install seam (#406): the damage-coefficient matrix (coeff[attackType]
# [armorType], thousandths) combat resolution multiplies each hit by. Wraps the

# slk gameplay-constant config. Until bound, every hit is dropped — so a world
# that wants damage seeds it here.

# Lua coroutine scheduler seam (#269/#270): the api-side hooks the luabind layer
# binds — register a resume continuation id and post a descriptive (serializable)

# deterministic green-thread model (execution-model.md §7), no JASS equivalent.
exact   Game.RegisterScriptCont
exact   Game.AfterScript

# Lua coroutine event-wait seam (#413): the api-side hooks the luabind WaitForEvent
# binds — register the single reserved-HandlerID script event dispatcher (at VM
# setup, before any load) and subscribe it to a kind so a coroutine can park on an
# EVENT (unit death, order, region enter), not only a timer. The kind→handler
# subscription is serialized sim state, so an event wait round-trips a save/load.
# Structural plumbing for the green-thread model (execution-model.md §7), no JASS
# equivalent (JASS waits are timer-only — TriggerSleepAction).
exact   Game.RegisterScriptEventDispatcher
exact   Game.SubscribeScriptEvent

exact   Game.HashSnapshot
exact   HashSystemNames
